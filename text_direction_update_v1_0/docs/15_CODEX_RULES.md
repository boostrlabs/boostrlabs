# 15 — Codex Rules

## Current task phase

For this update: text/documentation only.

## General rules

Codex should not:

- redesign frontend without explicit instruction;
- rename products;
- rewrite brand direction randomly;
- delete assets;
- move files without reason;
- change UI language;
- expose internal notes in partner UI;
- build all BOOSTR at once;
- implement payments/rewards/cashout before legal/technical scope;
- migrate SOLVE into core automatically;
- create public partner signup;
- create a self-service Linktree builder;
- convert BOOSTR into an agency/marketplace/disquera.

Codex should:

- work in small scoped tasks;
- preserve partner-specific assets;
- use manifests;
- update docs before code;
- keep mobile and desktop both working;
- maintain route isolation;
- mark outdated files instead of deleting;
- ask/flag when unclear.

## Codex after GitHub

Once GitHub is official, Codex should receive tasks such as:

- create app shell;
- implement workspace/persona switcher;
- implement partner configs;
- implement modular dashboard cards;
- implement Smart Link model;
- implement order model;
- implement guest/account rules;
- implement Rouvssen/Janko service purchase MVP;
- implement data engine base.
