# 02 — Product Architecture

## Core idea

BOOSTR is modular. Not every partner receives every module. Modules activate according to:

- plan;
- need;
- agreement;
- vertical;
- permissions;
- trust level;
- operational capacity;
- business reality.

## Core system areas

1. BOOSTR Core App
2. Partner Workspaces
3. Modular Dashboards
4. Smart Links / Deep Links
5. Partner Inventory
6. Checkout / Orders
7. Data Intelligence Layer
8. Follow-Up Engine
9. BOOSTR Network
10. Vertical Modules
11. BOOSTR Advisor Layer
12. Internal BOOSTR Manager Workspace

## Modular dashboard system

Do not create hardcoded dashboards per person as the primary architecture.

Create reusable cards/components that can be configured by partner:

- hero card;
- revenue card;
- fan/customer heat map;
- demand radar;
- what’s next;
- smart link preview;
- product/inventory card;
- follow-up queue;
- AI content suggestion;
- advisor note;
- order card;
- network recommendation;
- ticket/event card;
- beat store card;
- barber booking card;
- auto lead/deal card;
- proof of demand;
- smart crowd pricing;
- private post-purchase card.

Each partner has Dashboard Config:

- active cards;
- card order;
- theme;
- tone/copy rules;
- assets;
- modules active;
- mobile layout;
- desktop layout;
- permissions;
- visibility.

## Front doors

Partner websites may live on custom domains with fully custom branding. BOOSTR still powers identity, data, orders, and back-office.

Examples:

- `82ngel.com/account`
- `gemese.com/account`
- `jankodiorr.com/beats`

The user should feel they are inside the artist/partner world. BOOSTR appears as trusted infrastructure.

Microcopy:

- “Acceso seguro powered by BOOSTR Labs.”
- “Al continuar, creas una cuenta segura powered by BOOSTR Labs para acceder a tus compras, rewards y contenido de este partner.”
