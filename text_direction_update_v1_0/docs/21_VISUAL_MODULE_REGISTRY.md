# 21 — Visual Module Registry

## Purpose

Track visual modules without guessing from ZIP names.

## Rule

Do not rely only on file name to determine latest/best/official.

Track:

- Latest = newest by file/content/date.
- Best = strongest visual version.
- Official = approved/current or closest to user decisions.

If a newer version includes rejected text/design, it is not official.

## Required manifest fields

- module name;
- route;
- latest file;
- best file;
- official file;
- status;
- assets used;
- forbidden assets;
- user corrections;
- mobile status;
- desktop status;
- notes.
