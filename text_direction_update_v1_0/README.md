# BOOSTR Labs — Codex Text Direction Update v1.0

Date: 2026-07-07
Purpose: text-only direction pack for updating the local BOOSTR Labs folder before GitHub becomes the official source of truth.

## What this ZIP is

This is not the app, not the final frontend, not an asset pack, and not a backend implementation.

This ZIP is the current optimized strategic/documentation layer for BOOSTR Labs. It is meant to be given to Codex so Codex can update the local BOOSTR Labs folder's text, docs, roadmap, decisions, manifests, prompts, and direction files without programming.

## What Codex should do with it

Codex should:

1. Read `CODEX_START_HERE.md` first.
2. Update the local BOOSTR Labs folder with the current direction contained here.
3. Replace outdated text, old vision, old roadmap, old architecture assumptions, and bloated/irrelevant brainstorm docs.
4. Preserve local assets, HTML prototypes, screenshots, and frontend modules.
5. Create/update the required docs and manifests.
6. Do not code, do not build, do not install dependencies, do not redesign.

## What this ZIP does not include

It intentionally does not include heavy visual assets. The user has the real local assets in the Codex/local BOOSTR Labs folder. This pack documents how those assets should be classified and protected.

## Final workflow target

Local BOOSTR Labs folder → Codex applies this text update → user uploads full folder to GitHub → GitHub becomes source of truth → Cloudflare Pages deploys previews → ChatGPT reads GitHub for future planning/review → Codex builds scoped implementation tasks.
