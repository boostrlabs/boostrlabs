# BOOSTR Labs Front-End Roadmap After v0.2.4

## Current goal
Make every route feel like part of one ecosystem before Codex/backend begins.

## Done by this patch
1. Home becomes a network entry point.
2. Login becomes role-based shell.
3. App becomes client hub.
4. Admin becomes operator shell.
5. Partner pages become custom front doors.
6. Portfolio and OMG Beauty get connected.
7. Modules becomes QA matrix.

## Next 5 front-end builds
1. BOOSTR Home v2 with premium Apple/Tesla-coded visual identity and real conversion flow.
2. Admin OS v2 with tables, filters, lead cards, project pipeline and fake data closer to the real backend schema.
3. Client OS v2 with project detail page, file delivery, invoices and module activation states.
4. Partner OS v2 with referral dashboard demo, commission states and partner code flow.
5. 82 Store v2 with product pages, cart drawer, checkout intent, VIP unlock and drop timer.

## Codex/backend should wait until
- All primary links route internally.
- All major buttons have either a real destination or a clear demo placeholder.
- Admin/App/Login/Partner flows visually explain roles.
- Audit, Quote and Partner lead capture are unified conceptually.
