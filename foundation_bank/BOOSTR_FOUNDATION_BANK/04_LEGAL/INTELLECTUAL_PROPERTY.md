# Intellectual Property

## Core rule

BOOSTR retains ownership of the BOOSTR Core.

This includes:

- Reusable code.
- Frameworks.
- Templates.
- Internal methodology.
- Business logic.
- Architecture.
- Documentation.
- Component systems.
- Patterns.
- Automation logic.
- Non-client-specific tooling.

## Partner receives by default

Unless otherwise stated, a partner receives:

- Access to contracted BOOSTR modules.
- Use of their Smart Link / page / dashboard during the agreement.
- Final approved assets as defined in the SOW.
- License to use brand assets created for them, if included.
- Exportable files only if defined.

## Powered by BOOSTR

Powered by BOOSTR Labs is required by default on BOOSTR-built pages, Smart Links, and systems.

It can be removed only through:

- White-label fee.
- Buyout fee.
- Special contract.
- Enterprise agreement.

## Buyout / export / white-label

A partner may request to buy out, export, license, or white-label parts of their system.

BOOSTR may approve or reject this request.

If approved, the fee depends on:

- What is being exported.
- Whether BOOSTR Core code is involved.
- Whether the partner wants Powered by BOOSTR removed.
- Whether ongoing support is needed.
- Whether licensing is exclusive or non-exclusive.

## Important distinction

A client may own their brand/logo/assets but not automatically own the BOOSTR Core, reusable code, or methodology used to build their system.
